# CS-210


  Summarize the project and what problem it was solving.
This is the "Clocks" project  from CS 210 Module Three, the programs goal is to display two clocks simultaneously, a 12 hour and 24 hour one, and allow for 
the user to change the time and have this reflected accurately on both.

  What did you do particularly well?
I chose this project because I liked how modular everything was laid out, and I think I did a decent job keeping to that theme by making everything into it's
own file, and then tying it all together with a header file which I hadn't had to do before.

  Where could you enhance your code? How would these improvements make your code more efficient, secure, and so on?
I could improve the code by adding stronger input validation and making sure the program handles unexpected user input without failing.
There were a few ways I thought of that the user could input the wrong text and mess up the program, but I wasn't able to think of a clean way to solve for this.
I could also break up the program further, where some functions are doing similar things, and just have them reference a function dedicated to that specific task instead of reusing the code for say
the 12 hour clock vs the 24 hour clock.

  Which pieces of the code did you find most challenging to write, and how did you overcome this? What tools or resources are you adding to your support network?
When writing code I still find trouble with certain loops and making sure they aren't messy and are flowing right. I will overcome this by looking at examples I previously
did or examples online to remember the proper order. I have been compiling references to use like cheat sheets for setting up certain structures of loops that are commonly used
and trying to commit these to memory more and more.

  What skills from this project will be particularly transferable to other projects or course work?
Keeping everything modular, using different classes, and creating a header file to tie everything together will be really important going forward, so I'm glad I got experience specifically
with this project for my other work.

  How did you make this program maintainable, readable, and adaptable?
I made the program more maintainable and readable by separating different responsibilities into their own files and functions. I also used descriptive names and kept related code grouped together. 
Because the clock logic is separated from the menu and display logic, individual parts of the program could be changed without rewriting the entire project. This makes the program easier to update, debug, and adapt; 
a program like this is still small compared to an actual companies program, but the logic is still the same with regards to keeping things modular and easy to follow.
